# 설계 결정 문서

본 문서는 개발 과정에서 결정한 내용을 정리한 문서입니다.

## DB — mysql or postgresql 선택

postgresql의 JSONB 타입 컬럼이 프로젝션 기반 설계에 잘 맞다고 판단했습니다.

json 내부의 각 필드에 인덱스 설정이 가능하기 때문에, 새로운 타입의 이벤트가 추가되더라도 DB 정규화에 대한 고민을 줄이고 비즈니스 로직 구현에 집중 가능합니다.

json 내부의 각 필드는 외래키 설정이 불가능하다는 제약이 있지만, 외래키 미사용을 결정하였기 때문에 감수할 수 있는 트레이드오프라고 판단하였습니다.
아래 문단에서 관련 내용을 다룹니다.

## 외래키 미사용

- 이벤트 기반 설계에서 insert가 빈번하므로, 외래키 적용시 레코드 검증과정이 추가되어 오버헤드가 발생합니다.
- projection 방식 설계라는 점에서 event 테이블이 single source of truth 역할을 하고 있습니다. 따라서 foreign key 가 보호해야 할 진실이 없습니다.

## docker-compose 설정

postgresql 실행에 docker compose 를 이용합니다.
named volume 을 이용해서 영속화된 정보가 도커 볼륨에 저장되도록 구현했습니다.
호스트 os 와 디렉토리 경로를 연동하는 bind mount 방식과 구분되므로, 테스트 실행자의 로컬 기기 환경과 무관하도록 하여 편의성을 제공합니다.

## 인증 로직은 x-user-id 헤더로 대체

인증이 완료된 사용자의 id가 x-user-id 헤더에 주입된 상태라고 가정합니다.
